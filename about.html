The table added to the homepage.

Next, I created a price and package comparison table on the main page that scales with smaller devices well. I had a couple problems with the creation of this table, such as:

  • getting horizontal borders to render properly
  • stopping the table from collapsing on itself when on a narrow display
  • getting the first column to be wider than the rest

To get around these problems I:

  • set "border-collapse" to "collapse"
  • set "min-width" valuses for the table elements (in vw units)
  • set "width" in vw units for the first column

Guideline Summary
1.1.1 – Non-text Content

Provide text alternatives for non-text content
  • All images have a text alternative
  • There is no video or audio content to have descriptions for
  • All fields are text
1.2.1 – Audio-only and Video-only (Pre-recorded)

Provide an alternative to video-only and audio-only content
  • There is no video or audio content to provide an alternative for
  • In future (when music is available for streaming) I will add alternatives to the audio
1.2.2 – Captions (Pre-recorded)

Provide captions for videos with audio
  • This site does not contain any video to provide subtitles for
1.2.3 – Audio Description or Media Alternative (Pre-recorded)

Video with audio has a second alternative
  • This site does not contain any video to provide a second alternative for
1.3.1 – Info and Relationships

Logical structure
  • Each page is simple and not very long, each marked with a bold heading
  • All headings are inside of <h1> tags
  • Correct HTML Is used for page structure.
  • All HTML is validated using the W3C validator.
  • All forms have clear labels
1.3.2 – Meaningful Sequence

Present content in a meaningful order
  • All paragraphs are in order.
  • Headings are clear
  • All HTML is validated using the W3C validator.
1.3.3 – Sensory Characteristics

Use more than one sense for instructions
  • All areas of the website are simple to find when given instructions.
  • Elements all have more than one sense for instructions
    • (i.e. the white searchbox labelled "Search By Genre" on the left)
  • No elements rely on sound (this makes it hard to give instructions)
1.4.1 – Use of Colour

Don’t use presentation that relies solely on colour
  • No instructions rely on colour alone.
  • Other information (like charts and graphs) must not rely on colour alone.
1.4.2 – Audio Control

Don’t play audio automatically
  • The site does not contain any audio that plays automatically.
  • When audio is added (in future), there will be user controls so that the user can choose when to hear it.
2.1.1 – Keyboard

Accessible by keyboard only
  • All parts of the webiste are accessible through the use of the keyboard.
  • HTML is structured and validated properly which helps keyboard accessibility
  • There are no elements of the site that require timed keystrokes.
2.1.2 – No Keyboard Trap

Don't trap keyboard users
  • Upon testing, there are no area of the site that get a keyboard-only user trapped.
  • All elements can be navigated by just using a keyboard.
2.2.1 – Timing Adjustable

Time limits have user controls
  • There are no time controlled elements in this site.
  • If there was time controlled elements, I would have included an option to extend it
2.2.2 – Pause, Stop, Hide

Provide user controls for moving content
  • There is no moving content in this site.
  • If there was, I would have implemented controls for the user to stop, pause, hide etc.
2.3.1 – Three Flashes or Below

No content flashes more than three times per second
  • Nothing in this site flashes.
2.4.1 – Bypass Blocks

Provide a ‘Skip to Content’ link
  • For each page I have added a 'skip to content' link.
  • The link will take the user past the menu and to the content.
  • This allows users using screen readers the option to skip the repetitive menu.
2.4.2 – Page Titled

Use helpful and clear page titles
  • Each page has a clear and helpful name, displayed in larger text.
2.4.3 – Focus Order

Logical order
  • Everything on the site is laid out in an understandable way.
  • Tabbing through each page, the user will first tab through the menu, then the side content, then the main content (logical top to bottom, left to right)
2.4.4 – Link Purpose (In Context)

Every link’s purpose is clear from its context
  • All links are text, and provide a clear purpose from either their text or the content surrounding them.
  • The site has no links that are images or other media.
3.1.1 – Language of Page

Page has a language assigned
    The language of all pages is declared before the head of each page.
  • Language is set to "EN".

    3.2.1 – On Focus

    Elements do not change when they receive focus
    • No automatic actoions are performed when the user takes focus on an element.
    • Nothing automatically jumps.
    3.2.2 – On Input

    Elements do not change when they receive input
    • No automatic actoions are performed when the site recieves a user input.
    • Forms do not automatically jump once the user has filled them in. (irrelevant but the site does have one text entry at this stage)
    3.3.1 – Error Identification

    Clearly identify input errors
    • Due to the nature of the site, there is no change for the user to input something invalid, so this does not apply here.
    • In future verrsions, I would ensure that all fields are verified and give an explanation if they are invalid.
    3.3.2 – Labels or Instructions

    Label elements and give instructions
    • The only text input is labelled clearly.
    • No fields require a certain format.
    • No fields are compulsary.
    4.1.1 – Parsing

    No major code errors
    • The HTML has all been verified using the W3S Validator.
    4.1.2 – Name, Role, Value

    Build all elements for accessibility
    • As above, all code was checked for accessibility through the W3S validator.
